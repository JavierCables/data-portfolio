Summary

Goal

The primary goal of this project was to conduct a comprehensive analysis of the Netflix movies and TV shows dataset (8,807 records) to address 15 medium to advanced business problems using advanced SQL techniques. The project aimed to demonstrate proficiency in database setup, data import, complex querying, and data manipulation within PostgreSQL. Key objectives included calculating content distributions (movies vs. TV shows), identifying top actors and countries, analyzing genre popularity, and assessing content based on specific keywords in descriptions (e.g., 'kill' or 'violence'). The final deliverable was a fully documented project published on a GitHub repository.

Process

The project began by acquiring the Netflix dataset from Kaggle and initially exploring the data in Excel. A PostgreSQL database was set up using PG Admin 4, and the table schema was meticulously defined, carefully setting appropriate data types (e.g., VARCHAR limits) for all 8,808 potential rows. Data import required adjustments to column lengths, specifically for the 'listed in' column, utilizing DROP TABLE and CREATE TABLE commands to resolve import errors. Advanced SQL queries were then developed to solve 15 business problems, employing complex techniques such as Window Functions (RANK), various string manipulation functions (STRING_TO_ARRAY, UNNEST, SPLIT_PART, ILIKE), date conversion (TO_DATE), and subqueries.

Insights

The project successfully analyzed 8,807 Netflix content records, providing detailed SQL-based solutions to 15 complex business questions. Key analytical findings included content distribution (6,131 movies and 2,676 TV shows) and the most common content rating (TV-MA) for both types. Geographic analysis was enhanced by using UNNEST and STRING_TO_ARRAY to accurately split multi-country entries and identify top content-producing nations (like the US and India). Functional expertise was confirmed in handling non-standard text-based dates and durations. Finally, content was categorized (as 'Bad' or 'Good') based on description keywords using a CASE statement, demonstrating logical filtering capabilities. The complete analytical solution was documented and published on GitHub.
